import java.util.ArrayList;

interface ISubject {
    void register(IObserver iObserver);
    void remove(IObserver iObserver);
    void notifyInvestors();
}

interface IObserver {
    void update(double price);
}

class StockMarket implements ISubject {
    private ArrayList observers = new ArrayList<>();
    private double price;

    public double getPrice() { return price; }
    public void setPrice(double price) {
        this.price = price;
        notifyInvestors();
    }
    @Override
    public void register(IObserver iObserver) {
        observers.add(iObserver);
    }
    @Override
    public void remove(IObserver iObserver) {
        observers.remove(iObserver);
    }
    public void notifyInvestors() {
        System.out.println("Notifying investors about stock price change to: " + price);
        for (IObserver iObserver : observers) {
            iObserver.update(price);
        }
    }
}

class Investor implements IObserver {
    private String name;

    public Investor(String name) {
        this.name = name;
    }
    public void update(double price) {
        System.out.println("Investor : " + name + "received update. New stock price: " + price);
    }
}

public class Main {
    public static void main(String[] args) {
        StockMarket stock = new StockMarket();
        IObserver Tom = new Investor("Tom");
        stock.register(Tom);
        stock.setPrice(100);
        stock.remove(Tom);
    }
}

新設計有何優點?

在本題中，我們將原本的股市與投資者系統改寫為 Observer 設計模式，明確將 StockMarket 實作 ISubject，Investor 實作 IObserver，達到介面抽象與低耦合。透過這個設計，StockMarket 維護了一個觀察者清單，投資者可以自由註冊或移除，而當股價變動時，notifyInvestors 會自動呼叫所有觀察者的 update 方法通知最新價格。這樣的設計符合 介面隔離原則（Interface Segregation Principle） 與 依賴反轉原則（Dependency Inversion Principle），使股市與投資者之間的耦合度降低。

以生活化比喻來說，這就像遊戲中的「玩家追蹤系統」，玩家可以訂閱遊戲內的活動通知，當遊戲事件發生時，所有訂閱的玩家都會收到消息，而遊戲不需要知道每個玩家的細節。這種抽象設計的好處在於，如果未來要新增不同種類的投資者或通知方式，只需要實作 IObserver 介面即可，不必改動原本的股市程式碼，提高系統的擴充性與可維護性。

此外，Observer 模式使單元測試變得更簡單，我們可以對單個投資者或股市事件進行獨立測試，而不會干擾其他組件。若在大型專案中，這種設計還可以進一步加強，像是使用事件物件封裝通知資訊，或將觀察者容器改成線程安全（如 CopyOnWriteArrayList），以確保多執行緒環境下的資料一致性。

總結來說，透過 Observer 模式，我們成功實現了股市與投資者的低耦合、易維護與高擴充性。生活化的例子讓理解更直覺，但核心優點仍體現在技術層面：清晰的介面設計、可替換的觀察者、以及對未來擴充與測試的良好支援。!@#$

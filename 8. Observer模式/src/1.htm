import java.util.ArrayList;

interface ISubject {
    void register(IObserver client);
    void remove(IObserver client);
    void notifyInvestors();
}

interface IObserver {
    void update();
}

class StockMarket implements ISubject {
    private ArrayList observers = new ArrayList<>();
    private double price;

    public double getPrice() { return price; }
    public void setPrice(double price) {
        this.price = price;
        notifyInvestors();
    }

    @Override
    public void register(IObserver client) {
        observers.add(client);
    }
    @Override
    public void remove(IObserver client) {
        observers.remove(client);
    }
    @Override
    public void notifyInvestors() {
        System.out.println("Notifying investors about stock price change to: " + price);
        for (IObserver iObserver : observers) {
            iObserver.update();
        }
    }
}

class Investor implements IObserver {
    private StockMarket stockMarket;
    private String client;

    public Investor(StockMarket stockMarket, String client) {
        this.stockMarket = stockMarket;
        this.client = client;
    }

    public void update() {
        System.out.println("Investor :" + client + " received update. New stock price: " + stockMarket.getPrice());
    }
}

public class Main {
    public static void main(String[] args) {
        StockMarket stock = new StockMarket();
        IObserver Tom = new Investor(stock, "Tom");
        stock.register(Tom);
        stock.setPrice(100);
        stock.remove(Tom);
    }
}

新設計有何優點?

這個程式模擬了一個股票市場通知系統，使用了觀察者模式（Observer Pattern）。股票市場類別 `StockMarket` 代表主題，它可以註冊或移除投資人，並在股價變動時通知所有註冊的觀察者。投資人類別 `Investor` 實作了觀察者介面 `IObserver`，當收到通知時會印出最新股價。

在程式流程中，先建立股票市場實例，再建立投資人並註冊到市場。當股票價格透過 `setPrice()` 更新時，市場自動呼叫 `notifyInvestors()`，逐一通知所有投資人，投資人收到訊息後印出最新股價。如果投資人不想再接收通知，可以呼叫 `remove()` 取消註冊。

這種設計的好處是股票市場和投資人之間是鬆耦合的，新增或移除投資人不會影響市場的運作，也方便日後擴充，例如增加不同類型的投資人或加入更多市場訊息源。此外，這個模式與現實金融系統中的即時推播通知或訂閱消息系統相似，能直觀呈現事件驅動的設計思路。

interface MoveState {
    void action();
}

class Jumper implements MoveState {
    @Override
    public void action() {
        System.out.println("I am Jumping!");
    }
}

class Walk implements MoveState {
    @Override
    public void action() {
        System.out.println("I am Walking!");
    }
}

class Stop implements MoveState {
    @Override
    public void action() {
        System.out.println("I am Stopping!");
    }
}

public class Player {
    public MoveState moveState;

    public Player(MoveState moveState) {
        this.moveState = moveState;
    }
    public void setMoveState(MoveState moveState) { this.moveState = moveState; }
    public void move() {
        moveState.action();
    }
    public static void main(String[] args) {
        Player player = new Player(new Jumper());
        player.move(); // 顯示 I am Jumping!
        player.setMoveState(new Walk());
        player.move(); // 顯示 I am Walking!
        player.setMoveState(new Stop());
        player.move(); // 顯示 I am Stopping!
    }
}

新設計有何優點?
這份程式使用策略模式來設計角色的移動行為。

我先定義 MoveState 介面，把所有移動行為統一成 action() 方法，接著用 Jumper、Walk、Stop 分別實作跳躍、行走與停止三種不同動作。每個類別只負責自己的行為邏輯，彼此之間互不影響。

在 Player 類別中，並沒有直接寫死移動方式，而是透過建構子或 setMoveState 動態指定目前的移動狀態，呼叫 move() 時再委派給對應的策略執行。這樣可以在執行過程中隨時切換動作，例如從跳躍改為行走或停止。

這種設計的重點在於把「會改變的行為（移動方式）」獨立出來，使 Player 本身只負責控制流程，而不需關心具體動作內容。當未來需要新增新的移動方式時，只需新增一個實作 MoveState 的類別即可，不需要修改既有程式，提升了彈性與可維護性。!@#$

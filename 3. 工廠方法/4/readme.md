> ### 題目

想像一下，你正在創建一個物流管理應用系統 `(LogisticService)`，提供運送服務 `deliver(Object)` 。</br>
但是應用系統的第一個版本只能處理卡車運輸服務，因此大部分程式碼都放在 `Truck` 類別中，特別是提供 `transportCargo(Object)` 的卡車運輸服務。</br>
一段時間後，你的應用系統變得非常流行，每天都會收到來自海運公司的數十個請求，要求將海運物流整合到應用系統中。 </br>
但是目前的程式碼大部分都耦合到 `Truck` 類別。</br>
將 `Ship` 類別添加到應用系統中需要更改整個系統。 </br>
此外，如果以後決定向應用系統添加另一種交通方式，您可能需要再次進行所有這些更改。</br>
因此，請設計一個可以應付這種變化需求的物流管理服務應用系統。
